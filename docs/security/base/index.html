SpringBoot整合Security

@SpringBoot版本 : 2.0.4.RELEASE

应用环境

在我们开发Web应用程序的过程中，我们需要对我们的资源(URI)进行保护。我们常用的安全框架有Apache Shiro和Spring Security。Shiro我没有使用过，但是和Spring Security相比比较简单但是粒度也比较从粗。

在SpringBoot ，Spring Cloud大行其道的当下，Java开发已经等同于Spring开发了，Spring Security属于Spring的大家族一员，和自己的大哥整合的比较好，并且人家自己做的也不错啊。

Features

网上的教程都是很零散的Demo级别的教程，都是点级别的知识点，对于知道Spring Security详细细节的开发还好 ，如果初级入门 Spring Security的话就非常不友好，所以我想写这个Blog的时候遵循Arch-First(架构优先)的原则，先开启上帝视角来看Spring Security的运行原理，然后再深入进去修改每一个细节。

这个Blog是生产环境下使用的，动态的权限，动态的角色，不会像很多Demo中写的那样使用@PreAuthorize("hasRole('ADMIN')")这样子。

  • 基于数据库的动态权限认证
  • RBAC权限模型
  • Json返回，前后端分离
